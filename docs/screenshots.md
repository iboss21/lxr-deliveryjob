## ğŸ“¸ Screenshot Requirements

### Technical Specifications

| Specification | Value | Reason |
|--------------|-------|--------|
| **Resolution** | 1920x1080 (minimum) | Standard HD, widely viewable |
| **Format** | PNG or JPEG | Universal compatibility |
| **Quality** | High (PNG) or 90%+ (JPEG) | Professional appearance |
| **Aspect Ratio** | 16:9 | Standard gaming format |
| **File Size** | < 5 MB per image | GitHub/web friendly |
| **Naming** | Descriptive lowercase | Easy organization |

### Capture Settings (In-Game)

**Graphics Settings** (for best visuals):
```
Quality: High/Ultra
Weather: Clear or Cloudy (good lighting)
Time: Morning (7-9 AM) or Evening (5-7 PM) for golden hour
Post FX: Enabled
Anti-Aliasing: TAA High
```

**Camera Settings**:
- Disable UI (F7 or similar)
- Use cinematic camera where appropriate
- Show game context (landmarks, scenery)

## ğŸ¨ Capture Guidelines

### Do's âœ…

1. **Hide UI When Appropriate**
   - Press F7 (or your hide UI key)
   - Show UI for menu/notification shots

2. **Use Good Lighting**
   - Morning (7-9 AM in-game)
   - Evening (5-7 PM in-game)
   - Avoid night/fog unless intentional

3. **Choose Scenic Locations**
   - Recognizable landmarks
   - Beautiful Red Dead scenery
   - Clean backgrounds (not cluttered)

4. **Show Context**
   - Include minimap when relevant
   - Show player for scale
   - Display game elements clearly

5. **Capture High Quality**
   - Max graphics settings
   - High resolution
   - No compression artifacts

### Don'ts âŒ

1. **Avoid Bad Lighting**
   - âŒ Nighttime (unless moon is bright)
   - âŒ Heavy fog/rain (obscures details)
   - âŒ Backlit scenes (silhouettes)

2. **Don't Include**
   - âŒ Other players (unless intentional)
   - âŒ Irrelevant UI elements
   - âŒ Personal/server-specific branding (unless agreed)
   - âŒ Exploits or bugs

3. **Avoid Technical Issues**
   - âŒ Low FPS (choppy motion blur)
   - âŒ Pop-in visible
   - âŒ Floating objects/NPCs
